# Тестирование Яринг

## Запуск

```bash
npm run test:e2e:install
npm run test:e2e
```

`test:e2e` сам поднимает backend в режиме `E2E_TEST_MODE=1` и frontend. Если серверы уже запущены на `4000` и `3000`, Playwright переиспользует их.

## Публичные сценарии

- Главная: отображается hero, блок преимуществ, блок объектов, кнопки ведут в бронирование.
- Навигация: страницы `/`, `/booking`, `/rules`, `/contacts`, `/privacy`, `/login`, `/register` доступны; ссылки на `/prices` нет.
- Header: не перекрывает заголовки страниц.
- Footer: есть навигация, контакты, иконки VK/Telegram, карта.
- Контакты: карта Яндекса открывается с координатами `56.557858, 37.521514` и zoom `18.17`.
- Тема: переключатель меняет light/dark тему.
- Гость в бронировании: при попытке брони получает редирект на `/login` с сохранением выбранного объекта, например `/booking?entity=domik-u-ozera`.

## Пользовательские сценарии

- Регистрация: пользователь заполняет имя, телефон, пароль и переходит на шаг подтверждения телефона.
- Вход: подтверждённый пользователь входит по телефону/password и попадает в личный кабинет.
- Личный кабинет: отображаются данные пользователя и список броней.
- Посуточная бронь: создаётся бронь объекта daily, создаётся mock-оплата, после оплаты бронь видна в профиле со статусом `Подтверждено`.
- Почасовая бронь: создаётся бронь объекта hourly с временем, в профиле отображается диапазон времени.
- Отмена: пользователь отменяет свою подтверждённую бронь, статус меняется на `Отменено`.
- Выход: пользователь выходит из профиля и возвращается на публичную часть.

## Админские сценарии

- Guard: гость не может открыть `/admin`, его отправляет на `/login`.
- Вход админа: `+79990000000 / admin123`.
- Dashboard: админ видит разделы `Объекты`, `Брони`, `Страницы`.
- CMS объектов: админ создаёт объект, редактирует его, удаляет с подтверждением `confirm`.
- CMS страниц: админ редактирует `contacts`, `rules`, `privacy`; изменения видны на публичной странице.
- Брони: админ видит список пользовательских броней и их статусы.

## Что покрывают автотесты

- `tests/e2e/public.spec.ts`: публичные страницы, карта, тема, guest redirect.
- `tests/e2e/auth.spec.ts`: вход, ошибка входа, регистрация до шага кода.
- `tests/e2e/user.spec.ts`: профиль, daily/hourly booking через API, mock-оплата, отмена.
- `tests/e2e/admin.spec.ts`: guard, dashboard, CRUD объектов, CMS страниц, просмотр броней.

## Ограничения

- Полное подтверждение телефона через UI не проверяется, потому что код приходит через SMS-сервис или лог backend. Для e2e добавлен test-only endpoint `/api/auth/test-user`, доступный только при `E2E_TEST_MODE=1`.
- YooKassa проверяется через mock-оплату. Реальный redirect на YooKassa не должен запускаться в e2e.
